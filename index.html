About

Next-generation Anonymity

Tanegi is a decentralized P2P network with onion routing, written entirely in Rust. Unlike Tor, Tanegi uses post-quantum KEM (Kyber-1024) and a custom mathematical algorithm based on the 10958 number problem.

The Double Ratchet protocol (as in Signal) provides perfect forward secrecy in chat. Kademlia DHT organizes a distributed storage system with extended Reed-Solomon coding.

Beta version. The project is under development. Do not use for protecting critical data in production.

Privacy

Zero Leaks

Multi-layer protection against deanonymization at every level of the stack.

Kill Switch
iptables automatically blocks all traffic on VPN tunnel disconnect. Requires root. Protects against IP leaks.
Cover Traffic
Adaptive generation of dummy traffic (10% of bandwidth). Protection against traffic analysis.
Tracker Blocking
DNS blocking of google-analytics, doubleclick, facebook, Microsoft/Mozilla telemetry and 20+ other tracking domains.
DoH (DNS over HTTPS)
RFC 8484. All DNS queries are encrypted. Fallback — DNS via Tanegi circuit. No DNS leaks.
WebRTC Blocking
iptables rules block WebRTC STUN/TURN requests that could reveal the real IP address.

Quick Start

Up and running in 3 steps

1
Download and extract
Download tanegi.zip, extract to a convenient location.
unzip tanegi.zip && cd tanegi/
2
Make executable
The binary requires execution permissions and root privileges for iptables.
chmod +x tanegi && sudo ./tanegi
3
Configure SOCKS5 proxy
Configure applications (browser, curl) to use SOCKS5 proxy 127.0.0.1:9050.
curl --socks5-hostname 127.0.0.1:9050 https://check.torproject.org
+
Additional flags
Enable enhanced privacy protection.
sudo ./tanegi --tui --block-trackers --cover-traffic
Root privileges are required for kill switch and iptables rules (Linux). Some features are not available on macOS/Windows.
